### 写作约定

| 约定 | 说明 |
|------|------|
| G*N* 引用格式 | 跨页面引用通用规范编号，如"见 [G3](#g3-左侧菜单)" |

| 颗粒度 | 配置项级（清单每一行），不堆"模块挂菜单" |
| 不做什么 | 每节必须写明，并解释原因 |

# 第一部分：工作台（E02）

## 0. 范围

### 功能边界

| 包含 | 不包含 |
|------|--------|
| 多角色待办聚合展示 | 单角色深度操作页（5类E03首页） |
| 版本/届/角色三级联动筛选 | 成绩/评语/答辩安排等结果态页面 |
| 从工作台进入单角色首页 | 消息推送/IM即时通讯 |
| 当前页处理任务的抽屉表单 | 角色权限配置 |

### 不做什么（及原因）

| 不做的内容 | 原因 |
|-----------|------|
| 单角色深度操作页 | 那是E03的职责，工作台是聚合入口不是操作台 |
| 成绩/评语等结果态 | 工作台聚焦"待办"，已完成结果态不干扰主流程 |
| 消息推送/IM | 即时通讯是独立系统，不在本系统范畴 |
| 角色权限配置 | 那是后台管理系统的职责 |

---

## 1. 业务背景

工作台（E02）是多角色账号登录后的第一屏。一个账号可能同时持有多个角色（指导教师 + 评阅教师 + 系主任等），分布在不同培养类型版本和届次下。

**用户故事**：

- 作为多角色用户，我想登录后一眼看到所有待办任务的分布，不用逐个进入角色系统翻找。
- 作为多角色用户，我想按"培养类型版本 → 届 → 角色"三级快速筛选，聚焦当前需要处理的届次和角色。
- 作为多角色用户，我想在工作台直接处理部分任务（如审核），不用跳转页面。
- 作为多角色用户，我想快速进入某个角色的深度工作台继续操作。

### 3.2 培养类型版本块

纵向平铺当前账号绑定的所有培养类型版本。每个版本块结构：

```
版本名称                      该版本全部待办数
```

## 1. 业务背景

### 指导教师

指导教师是毕设过程中最核心的角色——全程参与选题、开题、中期、论文撰写、查重、答辩、归档 7 个阶段，每个阶段都有审核、指导、催办等具体任务。

**用户故事**：
- 作为指导教师，我想登录后一眼看到所有待处理任务，按紧急程度排序，不用逐个翻找学生。
- 作为指导教师，我想按"业务类型"（选题/开题/中期/论文等）快速筛选任务，聚焦当前阶段的工作。
- 作为指导教师，我想按"处理状态"（待提交/审核中/已完成/其他）筛选任务，区分"我需要催的"和"我正在审的"。
- 作为指导教师，我想点击某个学生就能看到他 7 个阶段的完整进展、当前卡在哪里、流程走到哪一步。
- 作为指导教师，我想在日历上看到所有学生的截止日期分布，提前安排审核计划。
- 作为指导教师，我想快速访问常用工具（维普慧导、学生管理、文件下载），减少导航跳转。

### 评阅教师

评阅教师负责在关键节点对学生的开题报告、中期报告、论文初稿/定稿、答辩稿等进行独立审阅和评分，不参与日常指导。

**用户故事**：
- 作为评阅教师，我想登录后一眼看到所有待审阅任务，按紧急程度排序。
- 作为评阅教师，我想按"业务类型"快速筛选，聚焦当前需要审阅的阶段。
- 作为评阅教师，我想按"处理状态"（待审核/审核中/已完成/其他）筛选。
- 作为评阅教师，我想在日历上看到所有审阅任务的截止日期分布。
- 作为评阅教师，我想快速访问常用工具（维普慧导、我的评分、文件下载）。

## 1. 业务背景

学生是毕设的执行者，全程参与从选题到答辩的各个环节，需要一个首页来：
1. 了解课题信息和当前所处阶段
2. 追踪各流程步骤的进展
3. 知道下一步要做什么、截止日期是哪天
4. 快速访问常用工具

### 用户故事

- 作为学生，我想登录后一眼看到我的课题信息，不用翻找。
- 作为学生，我想知道我现在处于哪个阶段、流程走到哪一步了。
- 作为学生，我想看到所有待办任务，知道要做什么、什么时候截止。
- 作为学生，我想在日历上看到关键日期分布，提前规划时间。
- 作为学生，我想快速访问维普慧导AI助手和文件下载等工具。

### 产品决策（已拍板）

| # | 决策 | 说明 |
|---|------|------|
| 1 | 流程+任务双视图互补 | 流程视图看"我处于哪个阶段"；任务视图看"我要做的具体事情"；每条任务带阶段标签 |
| 2 | 关键日期→日历样式 | 月度日历 + 截止日标记 |
| 3 | 项目信息卡3段固定展示 | 课题基本信息 / 评语与成绩 / 答辩组信息 |
| 4 | 维普慧导双入口保留 | Sidebar（左侧菜单）+ 右侧快捷工具 |
| 5 | 任务列表全是to-do | 不区分"做"vs"看"，统一为待办列表 |

### 4.1 标题行

标题"毕设流程" + "全部展开"/"全部收起"按钮。

**默认态**：已完成节点折叠，进行中节点展开，未开始节点折叠。

### 4.2 阶段节点

6阶段纵向排列：

```
1.选题 → 2.开题报告 → 3.中期检查 → 4.论文撰写 → 5.答辩 → 6.成绩与归档
```

每个节点为可折叠卡片。

**节点头部**：节点圆点（已完成/进行中/未开始）+ 阶段名称 + 状态标签 + 展开箭头（见 [G11](#g11-展开收起交互规范)）。

### 4.3 节点内容区（展开态）

从上到下依次展示：

#### ① 待办横幅（仅进行中节点且有任务时）

| 场景 | 文案 |
|------|------|
| 有待办 | `你的[任务名]待提交  距截止还有 N 天  [去提交]` |
| 无待办（等待审核） | "等待教师审核中" |
| 无待办（其他） | 不显示横幅 |

#### ② 流程步骤条

线性步骤序列（箭头连接），每步标记：
- ✓ 绿色圆点 = 已完成
- ● 蓝色圆点（脉动）= 当前
- ○ 灰色空心圆 = 等待

**各阶段步骤**：

| 阶段 | 步骤序列 |
|------|---------|
| 选题 | 提交选题 → 指导教师审核 → 系主任审核 → 审核通过 |
| 开题报告 | 提交开题报告 → 指导教师审核 → 系主任审核 → 审核通过 |
| 中期检查 | 提交材料 → 指导教师评分 → 系主任审核 |
| 论文撰写 | 初稿提交 → 指导教师评阅 → 修改定稿 |
| 答辩 | 答辩申请 → 答辩安排 → 答辩评分 |
| 成绩与归档 | 最终成绩评定 → 论文归档 |

#### ③ 子任务列表（仅当前阶段且有任务时）

每条含：
| 元素 | 说明 |
|------|------|
| 紧急标记 | 见 [G8](#g8-截止时间表达规范) |
| 任务名称 | 任务描述 |
| 截止信息 | 见 [G8](#g8-截止时间表达规范) |
| "去处理"按钮 | 跳转处理页 |

#### ④ 详情数据行

| 字段 | 空值 |
|------|------|
| 截止时间 | —（必有） |
| 提交时间 | "--" |
| 审核结果 | "--"（完成时显示结果） |
| 阶段状态 | 已完成 / 待提交 / 审核中 / 未开始 |

### 4.4 阶段状态判定

| 状态 | 条件 |
|------|------|
| 已完成 | 审核结果 = "通过" |
| 进行中 | 已提交或审核中，尚未全部通过 |
| 未开始 | 无任何提交记录 |

## 1. 业务背景

学校管理员需要监控全校的毕设进展，了解各学院/专业的任务分布和完成情况。

**用户故事**：
- 作为管理员，我想一眼看到所有待审核任务的数量和分布，不用逐个翻找。
- 作为管理员，我想知道全校的选题情况、提交进度、查重分布、评分分布等关键指标。
- 作为管理员，我想快速定位到某个工作流节点去处理任务。
- 作为管理员，我想看到关键日期分布，提前预警即将到期的事项。

## 3. 我的任务——工作流卡片网格

### 3.1 标题行

标题"我的任务" + 数字徽标。

**徽标规则**：
- 数字 = 所有卡片待处理数之和（不是卡片数量）
- 格式："我的任务 15"

### 3.2 卡片结构

2列网格布局。每张卡片：

| 元素 | 说明 |
|------|------|
| 紧急标记 | 见 [G8](#g8-截止时间表达规范) |
| 工作流名称 | 如"管理员审核指导关系" |
| 阶段标签 | 所属毕设阶段 |
| 优先级标签 | 紧急（≤2天）/ 一般 |
| 待处理数 + 截止时间 | 格式"待处理 N · 截止 YYYY-MM-DD"；未设时弱化"未设置" |
| 操作按钮 | "去处理"，跳转审核列表页 |

### 3.3 展开/收起

| 状态 | 规则 |
|------|------|
| 默认态 | 展示前4张 |
| >4张时 | 底部显示"展开更多（还有 N 条）↓" |
| 展开后 | 文案变为"收起 ↑"（见 [G11](#g11-展开收起交互规范)） |

### 3.4 空态

见 [G9](#g9-空态文案规范)："暂无待办任务"。
